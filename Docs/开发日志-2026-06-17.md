## 技术突破

### 1. BLE GATT 双服务架构

ESP32 同时运行 HID 服务（键盘）和 Profile 服务（数据接收）。
关键问题：两个服务不能各自注册 GATTS 回调，需要统一回调分发。

```c
static void unified_gatts_handler(event, gatts_if, param) {
    esp_hidd_gatts_event_handler(event, gatts_if, param);  // HID
    profile_gatts_handler(event, gatts_if, param);          // Profile
}
```

### 2. BLE GATT 128-bit UUID

Profile 服务使用 128-bit UUID 与 PC 端一致：
- Service: `12345678-1234-5678-1234-56789abcdef0`
- Char: `12345678-1234-5678-1234-56789abcdef1`

### 3. BLE 长写入处理

WinRT API 使用 BLE Long Write（Prepare Write + Execute Write），
ESP32 需要在 `is_prep=1` 时累积数据，`EXEC_WRITE` 时解析。

### 4. BLE 连接稳定性问题

BLE GATT 写入在 Windows 上不稳定（已配对设备连接路由问题）。
解决方案：改用串口（COM7）作为 Profile 数据通道，BLE 仅用于 HID 键盘。

### 5. GAP 广播修复

原版 `esp_hid_ble_gap_adv_start()` 不等待广播数据配置完成。
修复：添加 `ESP_GAP_BLE_ADV_DATA_SET_COMPLETE_EVT` 回调，配置完成后自动启动广播。
